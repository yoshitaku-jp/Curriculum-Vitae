## 職務経歴書

### 2014/04 ～ 現在: 株式会社ジール

職務：システムエンジニア

#### 小売業向けワークフローシステムの開発・サポート(2016.02 ~ )
- 担当:サーバーエンジニア
- 業務:基本設計・詳細設計・開発・テスト・サポート
- 技術スタック:JavaScript /

お客様の社内申請書の電子化に伴い、4,5名体制でお客様先に常駐しワークフローシステムの導入・開発・サポートをおこないました。お客様先にてウォーターフォール型の開発からアジャイル型への開発スタイルの変化があり戸惑うこともありましたが、開発サイクルが短くなった場合の立ち振る舞い・情報共有の大切さを感じることができました。

##### ワークフローシステムの画面開発・サポート
電子化対象となっている社内申請書が多いため開発のスピードが求められました。開発を楽にするためシステム側ですでにテンプレートとして提供されているアイテムもありましたが、お客様要望によりそれらをカスタマイズする場面もありました。ネット上には情報があまり共有されていないシステムだったため、既存アイテムの動作理解から地道な作業となりました。
開発スピードを上げるため、各申請書で共通しているモジュールを発見した場合は共通部品として切り出し開発メンバー全員が利用できるものとして共有しました。
また、技術支援サポートとしても業務に入っていたため、お客様先の若手エンジニアに対して技術情報共有・技術指導をおこないました。


#### 製造業向けワークフローシステムのデータ移行(2015.09 ~ 2016.1)
- 担当:サーバーエンジニア
- 業務:開発・テスト
- 技術スタック:JavaScript / WindowsServer2008 / SQL Server

お客様環境がオンプレミスからクラウドへ移行するにあたり、新環境ワークフローが正常に機能することやデータが移行されていることを確認・テストしました。また、バッチ処理の改修を任されました。

##### 新環境ワークフローの確認・テスト
先輩にレビューをしていただきながらテストケースの作成からテストまで基本的に一人でおこないました。エラーが発生した場合は、エラーが起こっただけではなく原因となる部分も含め報告し修正対応のサイクルを短くするように努めました。

##### バッチ処理の改修
夜間に起動するバッチ処理の改修をおこないました。ユーザ情報の更新や、組織の情報を更新するもので非常に大規模なデータを扱うものでした。テストでの動作確認では毎回全件おこなうと時間が足りなくなるため、データを細かく細分化し開発検証のサイクルを短くするように努めました。既存ソースコードの理解から問題部分の特定まで非常に困難でしたが、先輩に任せていただいたこともあり最後までやり遂げました。

#### Business Intelligenceツールのカスタマイズ(2014.07 ~ 2015.12)
- 担当:サーバーエンジニア
- 業務:基本設計・詳細設計・テーブル設計・画面設計・開発・テスト・検証
- 技術スタック:HTML / JavaScript / CSS / SQL / Windows Server 2012 R2 / SQL Server / Oracle Database 11g / 

お客様が利用していたBusiness Intelligenceツールのアップデートに伴い、5名前後の体制で新バージョンの製品カスタマイズ案件をおこないました。自身は先輩に指導していただきながら基本設計書や詳細設計書の作成に始まり、開発環境の構築やログイン画面の実装をおこないました。

##### 基本設計書・詳細設計書の作成
システムがどう動いているのを正とするかお客様と確認しながら基本設計書に起こしました。詳細設計書では自分自身やチームメンバがプログラミングに専念できるように細かく落とし込みました。実際にプログラミングをすると、詳細設計書には盛り込めていないことも多く頭で考えることと実際に手を動かすことの難しさを感じました。

##### ログイン画面の作成などSDKを用いた開発
別システムからBusiness Intelligenceツールへのログイン機能を実装しました。別システムから経由しない緊急時のログイン画面の作成や管理者用画面も含め、SDKを用い実装をおこないました。研修期間にローカル環境でプログラミング学習していたときとは違い、サーバー間でのやり取りやSDKの理解など考えることが多岐にわたりシステムを動かすことの楽しさと難しさを経験しました。
